# OfferLane

Affiliate / deals / coupon website built with **Next.js 15**. Product catalog and store coupon pages share one global shell (header, search, auth, footer).

**Handoff package:** [docs/handoff.md](docs/handoff.md) - **Client update guide:** [docs/client-update-guide.md](docs/client-update-guide.md) - **CMS import detail:** [docs/cms-import.md](docs/cms-import.md)

## 1. Project overview

**OfferLane** is an affiliate/deals/coupon siteâ€”not a single-product landing page or checkout funnel.

- **Inspired by** [ModernMavenLiving](https://www.modernmavenliving.com) for **structure and behavior** (catalog + store offers, outbound CTAs).
- **Brand, copy, colors, and assets** are **OfferLaneâ€™s own**â€”do not copy logos, text, images, or CSS from the reference site.
- **Conversion model:** outbound clicks and coupon reveal/copyâ€”not in-site cart or checkout (v1).

### Main page templates

| Template | Route | Purpose |
|----------|-------|---------|
| Product catalog | `/products` | Grid of affiliate products, pagination, **Shop Now** outbound |

Supporting content uses shared content pages from `public/data/pages.json`; blog posts are imported from `public/import/blogs.csv` into `pages.json`.

## 12. Pixel / ads tracking setup

Create `.env.local` (see `.env.example`):

```env
NEXT_PUBLIC_GA4_ID=
NEXT_PUBLIC_META_PIXEL_ID=
NEXT_PUBLIC_TIKTOK_PIXEL_ID=
```

| Behavior | Detail |
|----------|--------|
| Env **empty** | Third-party pixel scripts **do not load** |
| Env **set** | Scripts load from `components/analytics/AnalyticsScripts.tsx` |
| Always | `POST /api/track` and `trackEvent()` still run for internal logging |

UTM and ad click IDs (`utm_*`, `fbclid`, `ttclid`, `gclid`) are stored in **sessionStorage** and attached to all eventsâ€”see `lib/attribution.ts`.

**Debug:** In development, events log to the console. In production: `localStorage.setItem("debug_tracking", "1")` then reload.

## 13. Tracking events

| Event | When it fires |
|-------|----------------|
| `page_view` | Page load (includes UTM from session) |
| `view_store` | Store page load |
| `view_product` | Product card in viewport or click / Shop Now |
| `filter_offers` | All / Coupons tab |
| `sort_offers` | Latest / Popular / Expiry |
| `expand_offer` | Show More on offer |
| `get_coupon` | Show Coupon clicked |
| `copy_coupon` | Copy coupon code |
| `click_deal` | Get Deal clicked |
| `outbound_click` | Shop Now, Get Deal, Go to store |
| `paginate_products` | Products pagination |
| `newsletter_signup` | Footer or store inline form |
| `auth_modal_open` | Join us / Sign in |
| `add_to_cart` | **Reserved â€” not used in v1** |

Outbound flow: events fire first, then `navigator.sendBeacon` (fallback `fetch` keepalive) to `/api/track`, then redirectâ€”see `trackOutbound()` in `lib/analytics.ts`.
